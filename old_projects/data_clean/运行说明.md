# 运行脚本说明

## 问题：ModuleNotFoundError

如果遇到 `ModuleNotFoundError: No module named 'PIL'` 或 `ModuleNotFoundError: No module named 'pytz'` 等错误，说明 Python 解释器没有使用虚拟环境。

## 解决方案

### 方法1：在终端中激活虚拟环境后运行（推荐）

```bash
cd /Users/100buy/新链路/data_clean
source venv/bin/activate
python3 download_images.py    # 或 analyze_sales.py
```

### 方法2：直接使用虚拟环境的 Python

```bash
cd /Users/100buy/新链路/data_clean
./venv/bin/python3 download_images.py    # 或 analyze_sales.py
```

### 方法3：在 IDE 中配置 Python 解释器

如果使用 IDE（如 PyCharm、VSCode），需要配置使用虚拟环境的 Python：

**虚拟环境 Python 路径：**
```
/Users/100buy/新链路/data_clean/venv/bin/python3
```

**VSCode 配置：**
1. 按 `Cmd+Shift+P` (macOS) 或 `Ctrl+Shift+P` (Windows/Linux)
2. 输入 "Python: Select Interpreter"
3. 选择虚拟环境的 Python：`./venv/bin/python3`

**PyCharm 配置：**
1. File → Settings → Project → Python Interpreter
2. 选择虚拟环境的 Python 解释器

运行以下命令验证依赖是否安装：
